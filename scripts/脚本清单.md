# scripts 脚本清单

用于盘点 `scripts/` 下每个脚本的用途、引用关系与删除影响。  
**npm 发布流程**见 [`README.md`](README.md)；MCP 工具能力见 [`docs/ADA-MCP-接入手册.md`](../docs/ADA-MCP-接入手册.md) §2。

**图例**

| 建议 | 含义 |
|------|------|
| **保留** | 构建/发布/CI 或常用验证，删了会破坏流程 |
| **按需** | 有明确场景再用；不用可删 |
| **二选一** | 与同组另一脚本功能重叠，保留一个即可 |
| **可删** | 无 npm 引用、无被其他脚本 import；删前确认你不需要该场景 |

## 2. 构建与发布（4）

### `build-mcp-npm.mjs`

| 项 | 说明 |
|----|------|
| **功能** | 用 esbuild 打包 `apps/ada-mcp-server/dist/cli.cjs` 及 `plugins/driver-*.cjs`（playwright / appium / selenium），供 npm 发布。 |
| **谁在用** | `apps/ada-mcp-server` 的 `build:npm`、`prepublishOnly` |
| **典型命令** | `npm run build:npm -w @ada-mcp/mcp-server` |
| **删除影响** | 无法打出可发布的 mcp-server 包。 |
| **建议** | **保留** |

### `build-executable.mjs`

| 项 | 说明 |
|----|------|
| **功能** | 打包四入口可执行文件到 `release/`（ada-agent、ada-mcp、ada-web、ada-gui 等），含插件与 Windows 图标处理。 |
| **谁在用** | 根目录 `npm run build:exe` |
| **前置** | 需先 `npm install`、各 workspace build |
| **删除影响** | 无法生成 `release/*.exe`。 |
| **建议** | **保留**（若不做 exe 分发可考虑删，但会破坏 `build:exe`） |

### `generate-bundled-config.mjs`

| 项 | 说明 |
|----|------|
| **功能** | 从 `config/default.yaml` 生成内嵌配置 TS（bundled config），供 agent / mcp 打包使用。 |
| **谁在用** | `generate:bundled-config`；`apps/ada-agent`、`apps/ada-mcp-server` 的 `prebuild` |
| **删除影响** | prebuild 失败或配置过期。 |
| **建议** | **保留** |

### `restore-workspace-src-main.mjs`

| 项 | 说明 |
|----|------|
| **功能** | 将多个 workspace 包的 `package.json` 的 `main`/`types` 改回 `src/index.ts`，去掉 publish 相关字段；npm 发布后本地开发恢复用。 |
| **谁在用** | 无自动引用，维护者手动执行 |
| **删除影响** | 仅影响「发 npm 包后恢复 monorepo 开发态」的手工步骤。 |
| **建议** | **按需**（经常发 npm 则保留；从不发则可删） |

## 3. 仓库级测试（2）

### `e2e-smoke.mjs`

| 项 | 说明 |
|----|------|
| **功能** | 通过 `tsx` 调用 `ada-agent run`，跑 `tasks/web-real.tasks.json`；可选 `--with-appium` 再跑 `appium-real`；`--strict` 加 `--require-real`。 |
| **谁在用** | `test:e2e:smoke`、`test:e2e:smoke:strict`、`test:e2e:smoke:full` |
| **与 MCP 验证区别** | 走 **ada-agent 任务 JSON**，不是 MCP stdio 客户端。 |
| **建议** | **保留**（agent 端到端冒烟） |

### `verify-entrypoints.mjs`

| 项 | 说明 |
|----|------|

| **谁在用** | `test:entrypoints`（通常在 `build:exe` 之后） |
| **建议** | **保留**（若不做 exe 可与 `build-executable` 一并考虑移除） |

## 4. ada-mcp 验证（8 个脚本 + 1 依赖文件）

共同目标：在**不依赖 MCP Host UI** 的情况下，用命令行经 **stdio** 调 ada-mcp，验证 Web/App 真机流程。  
Python 脚本需：`pip install -r scripts/requirements-mcp-verify.txt`。

### `mcp-transport.mjs`（库脚本，不单独跑）

| 项 | 说明 |
|----|------|
| **功能** | 导出 `createAdaMcpTransport()`：封装 MCP 子进程启动方式 `dev`（tsx+源码）/ `local`（dist/cli.cjs）/ `npm`（pnpm dlx launcher）。 |
| **谁在用** | **`mcp-jd-app-verify.mjs`** `import` |
| **建议** | **保留**（删则 Node App 验证脚本要重写） |

### `mcp-bundled-smoke.mjs`

| 项 | 说明 |
|----|------|
| **功能** | 直连已构建的 `dist/cli.cjs`，`listTools` + `ada_health` + `ada_plugins`，确认 21 工具与三插件加载。 |
| **谁在用** | `test:mcp:bundled` |
| **前置** | 需先 `npm run build:npm -w @ada-mcp/mcp-server` |
| **建议** | **保留**（发布前必跑） |

### `ada_mcp_jd_verify.py`

| 项 | 说明 |
|----|------|
| **功能** | **Web / Chrome（Playwright）**：打开 www.jd.com → 等待 → Esc → 滚动 → 截图 → 关会话。默认 `--server` 在 npm script 里为 `dev`。 |
| **谁在用** | `test:mcp:browser`；`test:mcp:jd` 组合之一 |
| **建议** | **保留**（主 Web 验证） |

### `mcp_jd_firefox_verify.py`

| 项 | 说明 |
|----|------|
| **功能** | **本机 Firefox + 系统 Profile**，引擎 **Playwright**（`executablePath` + `userDataDir`）。打开京东 → 等待 → 截图。 |
| **谁在用** | `test:mcp:firefox` |
| **与 #14 关系** | 与 Selenium 版场景相同，实现路径不同。 |
| **建议** | **二选一**（只关心「系统 Firefox + 历史缓存」且接受 Selenium 时，可只留 #14） |

### `mcp_jd_firefox_selenium_verify.py`

| 项 | 说明 |
|----|------|
| **功能** | **本机 Firefox + 系统 Profile**，引擎 **Selenium**（`engine=selenium` + GeckoDriver）。流程同 Playwright 版。 |
| **谁在用** | `test:mcp:firefox:selenium` |
| **建议** | **二选一**（需验证 Selenium 插件时保留；否则可删） |

### `mcp-jd-app-verify.mjs`

| 项 | 说明 |
|----|------|
| **功能** | **京东 App 真机（Node）**：probe → adb 检查 → 可选装 mobile 依赖 → adb 启京东 → 截图 → 右滑×2 → 再截图。默认 `--server local`。 |
| **谁在用** | `test:mcp:app`、`test:mcp:app:probe`；`test:mcp:jd` |
| **前置** | Appium 常驻、真机、已 build `cli.cjs`（local 模式） |
| **建议** | **保留**（App 验证主脚本，Node 栈） |

### `mcp_jd_app_verify.py`

| 项 | 说明 |
|----|------|
| **功能** | 与 **`mcp-jd-app-verify.mjs` 流程等价**的 Python 实现（MCP Python SDK + stdio）。 |
| **谁在用** | `test:mcp:app:py` |
| **建议** | **二选一**（团队只用 Node 则可删 Python 版及 `test:mcp:app:py`） |

### `requirements-mcp-verify.txt`

| 项 | 说明 |
|----|------|
| **功能** | Python 验证唯一依赖：`mcp>=1.2.0`。 |
| **建议** | 有任一 `.py` 验证脚本则 **保留** |

## 8. 精简删除方案（参考）

按「只保留日常 MCP + 构建」最小集，可考虑删除下表项（**删前请改 `package.json` 对应 script**）：

| 可删候选 | 理由 |
|----------|------|
| `mcp_jd_app_verify.py` + `test:mcp:app:py` | 与 Node App 验证重复 |
| `mcp_jd_firefox_verify.py` **或** `mcp_jd_firefox_selenium_verify.py` + 对应 npm script | 两脚本同场景，留一个引擎即可 |
| `run-erp-jd-chrome-tasks.mjs` + `close-web-sessions.ts` + `run:erp-jd-chrome` | 专项 ERP 任务，与 MCP 验证无关 |
| `bootstrap-native-mock.mjs` + `bootstrap:native:mock` | 仅 mock 原生引导 |
| `restore-workspace-src-main.mjs` | 仅手工发 npm 后用 |
| `setup:gui-env`（npm） | 指向不存在的 `setup-tauri-env.ps1` |

**不建议删**：`build-mcp-npm.mjs`、`generate-bundled-config.mjs`、`mcp-bundled-smoke.mjs`、`mcp-transport.mjs`、`mcp-jd-app-verify.mjs`、`ada_mcp_jd_verify.py`（除非你完全不做 Web/MCP 验证）。

| 角色 | 建议保留的脚本 |

| **发布维护** | `build-mcp-npm.mjs`、`generate-bundled-config.mjs`、`mcp-bundled-smoke.mjs`、`restore-workspace-src-main.mjs`（可选） |
